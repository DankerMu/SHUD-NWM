# 国家水文预报系统 · 现状技术简报

> 最后更新：2026-06-14

---

## 1. 系统定位

一套**近实时、多流域的分布式水文预报业务系统**：以全球数值天气预报为气象强迫，驱动分布式水文模型 **SHUD**，逐时次滚动产出各流域河段流量过程（q_down）及衍生洪水频率产品，并通过 Web GIS 对外发布。

整个链路自动化、无人工干预滚动运行：**气象预报场 → 流域强迫场 → 水文模拟 → 河段流量 → 洪频分析 → 产品发布**。

---

## 2. 预报产出与覆盖口径

| 项目 | 当前口径 |
| --- | --- |
| 水文模型 | SHUD 分布式水文模型 |
| 气象强迫源 | NOAA **GFS** + ECMWF **IFS** 全球预报场（双源互补） |
| 强迫要素 | 降水、气温、相对湿度、风速、净辐射、气压（6 要素，插值至流域强迫站点） |
| 预见期 | **168 h（7 天）**；时间分辨率前 120 h 逐小时、120–168 h 3 h 步长（GFS） |
| 滚动频率 | 每个 GFS/IFS 发布时次（cycle）触发一轮 UTC 00时 和 12时|
| 主输出 | 河段流量过程线 q_down（入库 `hydro.river_timeseries`） |
| 已业务化流域 | **青海湖（QHH）、黑河（Heihe）**——端到端自动出预报、可展示 |
| 流域基础数据储备 | **10 个**（其余随自动化能力陆续纳入） |
| 衍生产品 | 洪水重现期 / 频率曲线（支线产品，见 §5） |

---

## 3. 业务化流程（数据流）

一次时次的完整链路：

> 气象预报场获取（GFS/IFS）→ 标准化为 canonical 气象产品 → 降尺度/插值生成**流域强迫场**（站点级、逐时）→ **SHUD 分布式模拟** → 河段流量过程入库 → 状态快照与 QC → **洪频/重现期分析** → 产品发布（Web GIS 瓦片 + 流量过程线）

**计算与展示分离**：模型运行、并行作业调度与建库在计算节点完成；对外展示（全国总览 + 单流域钻取）在独立的**只读节点**，两侧经发布产物解耦——展示侧的访问与故障不会影响后台计算与建库的稳定性。

---

## 4. 本阶段关键进展

- **多流域可扩展业务化**：由单一青海湖扩展到青海湖 + 黑河双流域并行；展示端改为**数据驱动**，新增流域无需改代码即可自动上图。
- **展示节点自治与自动上新**：展示节点自建本地库，自动发现/处理/展示计算端新产出的流域与时次（自动注册 → 强迫域镜像 → 输出解析 → 覆盖度物化），已在青海湖、黑河验证通过，正并入正式版本。
- **产品查询时延优化**：最新预报产品查询由约 **12–15 s 降至 < 1 s**（将河段/站点覆盖度等关键指标预计算物化，避免每次请求实时重算深层聚合）。
- **展示能力**：全国总览 + 流域钻取交互、河网渲染、天地图底图接入。

---

## 5. 当前科学口径与已知局限

- **洪水重现期 / 频率产品（支线）**：完整重现期分析依赖历史回算（hindcast）基线与质量汇总表；当前生产库的质量汇总表尚未建立、且各流域暂无 hindcast 基线，因此**重现期作为补充标注、不进入"产品就绪"判定**，也不阻断主流量预报展示。**主流量预报与展示不受影响**；打通需补齐历史回算基线与质量表。

- **数据源稳定性**：IFS（ECMWF）开放数据偶发访问限速（HTTP 503 / SlowDown），系统自动重试或降级到其他来源，一般不影响整体产出节奏。

> 总体：**主链路（强迫场 → SHUD → 河段流量 → 展示）稳定可用**；待办集中在洪频支线打通与河网/范围口径统一，均有明确方案，不阻断核心预报业务。

---

## 6. 下一步

1. **固化自动化**：将展示节点"自动上新"能力并入正式版本（收尾自动化测试与代码评审中）。
2. **扩流域**：把其余储备流域依次纳入自动业务化运行。
3. **打通洪频**：补齐质量汇总表与历史回算基线，产出完整重现期 / 洪水预警产品。
4. **持续提质**：稳定性、出图时延、河网与展示口径一致性。
