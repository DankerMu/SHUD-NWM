# 全国水文模拟系统：气象数据梳理与决策跟踪

**文档版本**：v0.2  
**更新时间**：2026-05-06  
**适用项目**：全国水文模拟系统  
**目的**：统一梳理本项目涉及的气象数据源，包括数据具体情况、下载渠道、权限状态、在系统中的用途、开发接入优先级和待决策事项，便于后续研发、运维和数据授权跟踪。

---

## 1. 结论摘要

全国水文模拟系统需要同时支持 **预报资料、再分析/融合实况资料、降水卫星资料、气象站观测资料** 四类气象数据。建议按以下角色分工：

| 数据角色 | 推荐主数据源 | 备用/扩展数据源 | 当前建议 |
|---|---|---|---|
| 未来 0–7 天预报 forcing | GFS | IFS Open Data | GFS 先做 MVP 主链路；IFS 作为并行预报场景接入 |
| 过去 0–7 天真实场 forcing | CLDAS-V2.0 RT/NRT | ERA5、GDAS、GPM IMERG | CLDAS 权限未解决前，先用 ERA5 + GFS/GDAS/IMERG 组合填补 |
| 历史长期 forcing | ERA5 | ERA5-Land、CLDAS、CMA-RA、CHIRPS | 用于率定、长期连续 analysis run、洪水频率样本构建 |
| 降水校验/补充 | CLDAS 降水 | GPM IMERG、CHIRPS、CMA 地面站 | 降水是洪水预报关键变量，建议单独做多源对比 |
| 气象站验证 | CMA 地面气象观测 | 全球交换站资料 | 用于 forcing 质控、偏差校正、模型评估，不建议作为唯一 forcing 来源 |

当前开发顺序建议：

```text
阶段 1：GFS → SHUD forcing → forecast run → 前端展示
阶段 2：ERA5 → analysis run → state_snapshot → forecast warm start
阶段 3：CLDAS adapter 预留 + 权限开通后切主真实场
阶段 4：IFS scenario 接入，支持 GFS/IFS 对比
阶段 5：GPM IMERG / CHIRPS / CMA 站点作为降水校验与偏差订正资料
阶段 6：ERA5-Land / CMA-RA 作为长期历史、对比和科研扩展资料
```

### 1b. 账号与下载通道策略

| 数据源 | 账号要求 | 生产主通道 | 备用通道 | 必须镜像 | 关键风险 |
|---|---|---|---|---|---|
| GFS | 不需要 | AWS Open Data / NOAA Big Data | NOMADS filter、NCEP 产品服务器、NCEI 归档 | 是 | 等待 f000–f168 完整发布 |
| IFS Open Data | 不需要账号，需遵守 ECMWF ToU | ECMWF Open Data / ecmwf-opendata client | AWS / Azure / Google 公共云镜像 | 是 | 仅保留最近约 12 个周期；06/18 UTC 到 144h |

## 2. SHUD 对气象 forcing 的需求

SHUD 输入数据中，气象驱动属于高必要性时间序列数据，原始资料要求为“小时～日”的降雨、气温、湿度、辐射、风速、气压。SHUD 的 `.tsd.forc` 文件用于记录气象站点数量、开始日期、forcing 文件目录和每个 forcing CSV 文件名；forcing CSV 主要列为：

| SHUD 列名 | 含义 | SHUD 文档单位/范围 | 系统内部标准单位 | 备注 |
|---|---|---:|---:|---|
| `Day` | 相对开始日期的日数 | day | day | 小时数据用 1/24 递增 |
| `PRCP` | 降雨 | mm/day | mm/day | 若源数据为小时累计 mm，应换算为强度 mm/day；若配置改为日尺度，则按日累计处理 |
| `TEMP` | 气温 | °C | °C | GFS/IFS/ERA5 常为 K，需减 273.15 |
| `RH` | 相对湿度 | 0–1 | 0–1 | 可由露点温度、比湿、气压反算 |
| `wind` | 风速 | m/s | m/s | 由 10m U/V 合成时为 `sqrt(u²+v²)` |
| `Rn` | 太阳净辐射 | W/m² | W/m² | 不同资料的“净辐射/短波辐射/累积辐射”语义不同，需逐源确认 |
| `Press` | 气压 | Pa，可缺省 | Pa | 建议始终提供，减少区域间差异 |

**开发要求**：所有数据源最终都应转换到统一的 `Canonical Meteorological Product`，再由 forcing 生成器映射到 SHUD forcing 文件。不要让 SHUD 运行任务直接读取 GRIB/NetCDF 原始数据。

## 4. 数据源详细卡片

### 4.1 GFS：Global Forecast System

#### 4.1.1 数据具体情况

GFS 是 NOAA/NCEP 的全球数值天气预报系统，包含气温、风、降水、土壤湿度、臭氧等多类大气和陆面变量。NCEI 页面说明 GFS 每天 4 个模式周期，即 00、06、12、18 UTC；NOAA/NCEP 产品清单中，0.25° GFS `pgrb2.0p25` 文件名形如 `gfs.tCCz.pgrb2.0p25.fFFF`，其中 `CC` 是起报时次，`FFF` 是 000–384 小时预报时效。

项目推荐使用：

```text
产品族：GFS 0.25° pgrb2
周期：00 / 06 / 12 / 18 UTC
预报时效：f000–f168，覆盖未来 7 天
格式：GRIB2
空间范围：全球，项目中裁剪中国及周边区域
```

#### 4.1.2 下载渠道

推荐按优先顺序使用：

1. **AWS Open Data / NOAA Big Data Program**  
   适合云端批量下载和对象存储镜像。NCEI 说明 NOAA Big Data Program 提供 0.25°/0.5° 分析和预报资料的 trailing 30-day window。

2. **NOMADS HTTPS filter**  
   适合只下载指定变量、指定层、指定中国区域，节省带宽。

3. **NCEP 产品服务器 FTP/HTTPS**  
   适合作为 NOMADS 异常时的备用渠道。

4. **NCEI 历史归档/HAS/TDS**  
   适合后续历史回补，不建议作为实时主下载入口。

示例路径模板：

```text
NOMADS 原始文件：
https://nomads.ncep.noaa.gov/pub/data/nccf/com/gfs/prod/gfs.YYYYMMDD/CC/atmos/gfs.tCCz.pgrb2.0p25.fFFF

AWS S3：
s3://noaa-gfs-bdp-pds/gfs.YYYYMMDD/CC/atmos/gfs.tCCz.pgrb2.0p25.fFFF
```

#### 4.1.3 在项目中的用途

| 用途 | 使用方式 |
|---|---|
| 未来 7 天 forecast forcing | 主预报源，先接入 GFS deterministic scenario |
| `forecast_gfs_deterministic` 场景 | 每个 GFS 起报周期单独生成 scenario |
| 前端气象格点图层 | 展示未来有效时刻降雨、气温、风等格点场 |
| SHUD forcing 代站 | 通过预计算权重将 GFS 网格提取到每个流域 forcing station |
| CLDAS 权限未解决时的临时补位 | 最近 0–5 天若 ERA5 不可用，可用 GFS analysis/short forecast 作为 best_available 低优先级补位 |

#### 4.1.4 变量映射建议

| SHUD forcing | GFS 变量候选 | 转换说明 |
|---|---|---|
| `PRCP` | `APCP` / precipitation accumulation | 累计降水转时段降水，再转为 mm/day 强度 |
| `TEMP` | 2m temperature | K → °C |
| `RH` | relative humidity / specific humidity + pressure / dewpoint | 优先直接取近地面 RH；否则由比湿、气压、温度反算 |
| `wind` | 10m U/V wind | `sqrt(u²+v²)` |
| `Rn` | net shortwave/longwave 或上下行辐射 | 优先净辐射；否则由上下行短波/长波合成 |
| `Press` | surface pressure | Pa |

#### 4.1.5 当前决策

```text
状态：enabled for MVP
优先级：P0
决策：作为首个业务化预报数据源。
待办：确认中国区裁剪 bbox、变量过滤清单、f000–f168 下载步长、累计降水解析规则。
```

### 4.2 IFS Open Data：ECMWF 开放预报子集

#### 4.2.1 数据具体情况

ECMWF Open Data 提供 IFS 和 AIFS 实时预报资料的免费开放子集。Open Data 页面说明，产品一般为 0.25°、GRIB2；IFS 高分辨率产品在 00/12 UTC 起报时，step 为 0–144 小时每 3 小时、150–360 小时每 6 小时；06/18 UTC 起报时，step 为 0–144 小时每 3 小时。Open Data 采用滚动归档，只保留最近 12 个预报周期，约 2–3 天，因此项目必须及时镜像。

项目推荐使用：

```text
产品族：IFS Open Data high-resolution products
周期：00 / 06 / 12 / 18 UTC
分辨率：0.25°
格式：GRIB2
00/12 UTC：可满足未来 7 天
06/18 UTC：仅到 144h，不能完整覆盖未来 7 天
```

#### 4.2.2 下载渠道

推荐渠道：

1. **ECMWF Open Data Portal**：`data.ecmwf.int`
2. **ecmwf-opendata Python client**：适合调度器批量下载
3. **AWS / Azure / Google Cloud 复制数据**：ECMWF 页面说明开放资料在主要公共云有复制数据，可用于冗余和提高访问稳定性。

示例 Python 接入方式：

```python
from ecmwf.opendata import Client

client = Client(source="ecmwf")  # 可切换 source="aws" / "azure" / "google"
client.retrieve(
    date="20260501",
    time="00",
    step=[0, 3, 6, 9, 12, 24, 48, 72, 96, 120, 144, 150, 156, 162, 168],
    stream="oper",
    type="fc",
    levtype="sfc",
    param=["2t", "2d", "10u", "10v", "tp", "sp", "ssr", "str"],
    target="ifs_20260501_00_sfc.grib2",
)
```

#### 4.2.3 在项目中的用途

| 用途 | 使用方式 |
|---|---|
| 未来 7 天 forecast forcing | 与 GFS 平行生成 `forecast_ifs_deterministic` 场景 |
| 多源预报对比 | 前端河段曲线同时展示 GFS 和 IFS |
| 不确定性表达 | 若 GFS/IFS 对同一河段洪峰差异大，标记预报不确定性 |
| best_available forecast | 00/12 UTC IFS 可作为未来 7 天高质量候选源 |

#### 4.2.4 变量映射建议

| SHUD forcing | IFS Open Data 变量候选 | 转换说明 |
|---|---|---|
| `PRCP` | `tp` total precipitation | 累积量转时段量；单位需转 mm |
| `TEMP` | `2t` | K → °C |
| `RH` | `2t` + `2d` | 由气温和露点温度计算 RH |
| `wind` | `10u`, `10v` | `sqrt(u²+v²)` |
| `Rn` | `ssr` + `str` 或 `ssrd` + `strd` | 累积辐射量转 W/m²，注意 step 差分 |
| `Press` | `sp` | Pa |

#### 4.2.5 当前决策

```text
状态：enabled/planned
优先级：P1
决策：作为 GFS 的并行预报场景，不与 GFS 混合入库。
待办：制定 06/18 UTC 只有 144h 时的前端展示规则：标记“仅 6 天”或用最近 00/12 UTC 补齐。
```

### 4.3 CLDAS-V2.0 RT/NRT：中国气象局陆面数据同化系统

#### 4.3.1 数据具体情况

CLDAS-V2.0 是中国气象局陆面数据同化系统产品。实时产品说明其覆盖亚洲区域 0–65°N、60–160°E，空间分辨率 0.0625° × 0.0625°，时间分辨率 1 小时；包括大气驱动场产品、地表温度、土壤湿度、土壤温度和土壤相对湿度等。实时产品滞后约 1 小时更新发布。近实时产品检索页面显示时间分辨率支持 1 小时和 1 天，要素包括 2m 气温、2m 比湿、10m 风速、地面气压、小时降水、短波辐射、土壤湿度、地表温度、土壤温度和土壤相对湿度等。

项目推荐使用：

```text
产品族：CLDAS-V2.0 RT / NRT
空间范围：0–65°N, 60–160°E
空间分辨率：0.0625°
时间分辨率：1 小时优先
核心变量：2m 气温、2m 比湿、10m 风速、地面气压、小时降水、短波辐射
权限状态：restricted，需注册和权限确认
```

#### 4.3.2 下载渠道

当前可确认渠道：

1. **气象科学专业知识服务系统 / 中国气象数据网**  
   数据代码：
   - `NAFP_CLDAS2.0_RT`：CLDAS-V2.0 实时产品
   - `NAFP_CLDAS2.0_NRT`：CLDAS-V2.0 近实时产品

2. **注册/登录后检索下载**  
   页面显示共享级别为普通注册用户访问；但批量下载、API、自动化下载权限需要在实际账号开通后确认。

3. **后续需确认**  
   是否支持 API token、批量订单、脚本化下载、历史归档下载、变量级订阅、单位和文件格式说明文档获取。

#### 4.3.3 在项目中的用途

| 用途 | 使用方式 |
|---|---|
| 过去 7 天真实场 forcing | 中国区优先真实场资料，生成 `analysis_true_field` |
| forecast 初始状态 | 用 CLDAS 驱动 SHUD 连续 analysis run，保存 `.cfg.ic` state snapshot |
| best_available 气象图层 | 有 CLDAS 时，最近时段优先显示 CLDAS |
| 降水质量控制 | 与 GFS/IFS/IMERG/地面站降水对比 |
| 土壤状态参考 | 土壤湿度、土壤温度可用于模型状态诊断，不直接替代 SHUD 状态 |

#### 4.3.4 变量映射建议

| SHUD forcing | CLDAS 变量 | 转换说明 |
|---|---|---|
| `PRCP` | 小时降水 | 小时累计 mm → mm/day 强度；或按配置做日累计 |
| `TEMP` | 2m 气温 | 确认单位，通常按 K/°C 标准转换 |
| `RH` | 2m 比湿 + 地面气压 + 温度 | 由比湿反算 RH |
| `wind` | 10m 风速 | 可直接使用；若有 U/V 则合成 |
| `Rn` | 短波辐射 | 需确认是下行短波、净短波还是时段累计；若非净辐射，需要经验公式或反照率处理 |
| `Press` | 地面气压 | 转 Pa |

#### 4.3.5 当前决策

```text
状态：restricted
优先级：P0，但受权限制约
决策：架构上必须预留 CLDAS adapter；权限未解决前先用 mock + ERA5/GFS/GDAS/IMERG 组合替代。
待办：解决账号权限、批量下载、API 文档、变量单位、文件格式、服务稳定性和许可条款。
```

### 4.4 ERA5 Single Levels：ECMWF/Copernicus 全球再分析

#### 4.4.1 数据具体情况

ERA5 是 ECMWF 第五代全球再分析资料，CDS 页面说明它从 1940 年至今提供全球小时资料；单层资料为规则经纬度网格，水平分辨率 0.25° × 0.25°，文件格式为 GRIB。ERA5 每日更新，但约有 5 天迟滞；早期发布的 ERA5T 在 2–3 个月后可能被最终资料替换。

项目推荐使用：

```text
产品族：ERA5 hourly data on single levels
数据集名：reanalysis-era5-single-levels
空间分辨率：0.25°
时间分辨率：hourly
时间范围：1940–present
格式：GRIB
更新：daily，约 5 天迟滞
```

#### 4.4.2 下载渠道

推荐渠道：

1. **Copernicus Climate Data Store Web Portal**
2. **CDS API**
3. **earthkit / CDS 工具链**

示例 CDS API 变量：

```python
import cdsapi

c = cdsapi.Client()
c.retrieve(
    "reanalysis-era5-single-levels",
    {
        "product_type": "reanalysis",
        "variable": [
            "2m_temperature",
            "2m_dewpoint_temperature",
            "10m_u_component_of_wind",
            "10m_v_component_of_wind",
            "surface_pressure",
            "total_precipitation",
            "surface_net_solar_radiation",
            "surface_net_thermal_radiation",
        ],
        "year": "2026",
        "month": "04",
        "day": ["20", "21", "22"],
        "time": ["00:00", "01:00", "02:00", "03:00"],
        "area": [55, 70, 15, 140],
        "format": "grib",
    },
    "era5_20260420_22_china.grib",
)
```

## 6. 统一中间产品格式

所有原始气象数据进入系统后，应转换成统一中间产品：

```text
Canonical Meteorological Product
  ├── source
  ├── source_version
  ├── cycle_time
  ├── issue_time
  ├── valid_time
  ├── lead_time_hours
  ├── variable
  ├── value
  ├── unit
  ├── grid_definition
  ├── spatial_resolution
  ├── temporal_resolution
  ├── processing_level
  ├── quality_flag
  ├── object_uri
  └── checksum
```

### 6.1 变量标准化命名

| 标准变量 | 含义 | 标准单位 |
|---|---|---|
| `precip_rate` | 降水强度 | mm/day |
| `precip_amount` | 时段降水量 | mm |
| `t2m` | 2m 气温 | °C |
| `rh2m` | 2m 相对湿度 | 0–1 |
| `q2m` | 2m 比湿 | kg/kg |
| `wind10m` | 10m 风速 | m/s |
| `u10m` | 10m U 风 | m/s |
| `v10m` | 10m V 风 | m/s |
| `surface_pressure` | 地面气压 | Pa |
| `net_radiation` | 净辐射/太阳净辐射 | W/m² |
| `shortwave_down` | 下行短波辐射 | W/m² |
| `longwave_down` | 下行长波辐射 | W/m² |

### 6.2 单位转换规则

| 转换项 | 规则 |
|---|---|
| 温度 | K → °C：`T_C = T_K - 273.15` |
| 降水 | kg/m² ≈ mm；m → mm：`mm = m × 1000` |
| 累计降水 | 对相邻 valid_time 做差分；遇到起报周期重置需从 f000 重新计算 |
| 降水强度 | `PRCP_mm_day = amount_mm / duration_day` |
| 风速 | `wind = sqrt(u10² + v10²)` |
| 相对湿度 | 优先读取 RH；否则由气温、露点或比湿反算 |
| 辐射 | 累积 J/m² → W/m²：`W/m² = ΔJ/m² / Δseconds` |
| 气压 | hPa → Pa：`Pa = hPa × 100` |

## 9. 质量控制与监控指标

### 9.1 下载质量

| 指标 | 要求 |
|---|---|
| 文件完整性 | checksum 或 size 校验 |
| 周期完整性 | 每个 cycle 应包含目标 forecast hour 列表 |
| 变量完整性 | forcing 所需变量必须齐全，否则标记 partial |
| 时间连续性 | valid_time 不得缺测；允许记录 gap |
| 空间覆盖 | 必须覆盖中国及目标流域 bbox |

### 9.2 物理范围检查

| 变量 | 检查规则 |
|---|---|
| 降水 | 不得为负；极端值需标记但不自动删除 |
| 气温 | 中国区极端范围可先设 -60°C～60°C |
| 相对湿度 | 应在 0–1，超过范围需裁剪或标记 |
| 风速 | 非负；极端大风标记 |
| 气压 | 中国区地面气压按地形差异检查，异常值标记 |
| 辐射 | 夜间短波应接近 0；负净辐射需确认语义 |

### 9.3 与站点/多源对比

建议对每个流域维护每日 QC 报告：

```text
- 流域平均降水：GFS vs IFS vs CLDAS vs IMERG vs CHIRPS
- 站点气温 RMSE / bias
- 站点降水命中率、漏报率、虚警率
- forcing station 时间序列缺测率
- 各数据源迟滞统计
- 数据源替换记录：ERA5T → ERA5 final
```
