## 3. Grid Snapshot Registration

- [ ] 3.1a Define the registry input record: the canonical `grid.json` under `canonical/{source}/grid/{grid_id}/grid.json` (source of `schema_version`, `grid_id`, `layout`, `axis_order`, `shape`, `longitudes`, `latitudes` — see `workers/canonical_converter/converter.py:1824-1831`) merged with:
  - `converter_version` — read from `workers.canonical_converter.converter.expected_converter_version(source_id)` (see `converter.py:387-400`, backed by per-source `CanonicalConverterConfig().converter_version` string).
  - `native_resolution` — derived from `grid.json` axis spacing (median absolute difference of consecutive `longitudes` and `latitudes`, asserted equal for a rectilinear grid); if `grid.json` layout is non-rectilinear, registration fails closed with a structured error requiring the operator to extend the canonical converter.
  - `latitude_order` — computed from the raw `grid.json` `latitudes` array (`ascending` if strictly increasing, `descending` if strictly decreasing; otherwise fail closed).
  - `flatten_order` — fixed to `y_major_lat_then_lon` matching the deterministic nested-loop iteration `(lat, lon) for lat in latitudes for lon in longitudes` at `workers/forcing_producer/producer.py:1470-1479` inside `_grid_points_for_dataset` (defined at `producer.py:1381`); the block emits `GridPoint(grid_cell_id=str(index), longitude=_normalize_longitude(longitude), latitude=latitude)` — i.e. y outer / x inner, index-as-string as the `grid_cell_id`; recorded explicitly on the snapshot. This is the same iteration order that `grid_signature_hash` hashes against; the registry's `canonical_ordinal` field records the 1-based counterpart of the producer's 0-based `index`.
  - `valid_from` / `valid_to` — sourced from a new sidecar file `canonical/{source}/grid/{grid_id}/grid_snapshot_metadata.json` (a Task 3.1a artifact this change introduces) carrying `{"valid_from": ISO8601, "valid_to": ISO8601 | null, "download_bbox": {south, north, west, east}}`; if the sidecar is missing, registration fails closed instructing the operator to author it before backfill.
  - `grid_cell_id` — generated at read time as `str(index)` where `index` iterates `0..expected_count-1` in the fixed flatten order (matches `workers/forcing_producer/producer.py:1462` behavior; producer semantics are not changed by this task).
