## Why

The direct-grid migration (docs/ForcingReplace §15/§16) sequences the first single-source pilot *after* platform readiness, grid registry, and the route/state/display switch mechanisms land — but three enablement gaps block a pilot from *building and producing* trusted direct-grid assets:

1. **Dangling `z_policy` authority.** `openspec/specs/direct-grid-binding-artifact/spec.md` pins station `z` to "the `z_policy` approved by the solver audit in change `cmfd-direct-grid-platform-readiness`". That solver audit was intentionally descoped from readiness (Epic #886 proposal: *"Solver forcing-consumer auditing is intentionally out of scope"*) and never produced a verdict — the spec points at nothing. Without an authoritative verdict the mapping builder's `ZPolicy` input has no source of truth.
2. **Missing operator entrypoint (proposal/design ↔ code drift).** The mapping builder (Epic #909) shipped library-only: `forcing-mapping-asset-build` `proposal.md:28` and `design.md:45` both name `workers/mapping_builder/cli.py` as the entrypoint, but no task ever created it. There is no end-to-end command that chains the existing G0–G5 library stages into one offline build.
3. **Unwired bbox guard.** `packages/common/grid_registry_bbox_guard.py::verify_download_bbox_matches_registry` exists and is unit-tested but is called by nothing; `canonical-source-grid-registry` Task 3.2 explicitly defers "producer preflight" wiring to a follow-up change. A deployment `NHMS_DOWNLOAD_BBOX_*` drift can still silently move `grid_cell_id`s under a registered snapshot.

This change closes all three so a pilot can build and produce direct-grid assets on an authoritative, fail-closed footing. It activates **no** production basin (§16 ordering: mechanism only; activation happens after pilot/canary).

## What Changes

- Add a narrow, written `z_policy` solver-audit **verdict** — answering exactly three questions against pinned SHUD solver commit `3aec65755926c478e13ca7d4fea80715e4e90345`: (a) does the `.tsd.forc` read chain read the station-row `Z`, (b) what is it used for, (c) is there a `.cfg` correction switch. The verdict evidence file is committed to this change's `evidence/` and becomes the authoritative `z_policy` input for the mapping builder's binding stage — resolved through pinned constants (expected verdict value + the file's SHA-256) in a new `workers/mapping_builder/z_policy_verdict.py`, which also implements the deterministic `per_cell_z` sampler `nearest_mesh_node_elevation_v1`; post-archive the file lives under the archive relocation of this change directory, and the pinned checksum (not the path) anchors authority. Verdict recorded: `model_dem_at_cell_center` (station `Z` participates in temperature lapse correction, so an explicit elevation source is required; `sentinel` is rejected).
- Add `workers/mapping_builder/cli.py`: an offline operator entrypoint that chains the existing library stages (G0/G1 integrity → algorithm → rewrite → binding → evidence) into one end-to-end build. Input authority is the object-store release-frozen package shape (`<object-store-root>/models/basins_<basin>_shud/<release>/package/`; root default `/home/ghdc/nwm/object-store`, changeable only via an explicit evidence-recorded `--object-store-root` — the sanctioned test channel), never the `Basins` dev workspace; a path of neither shape fails closed. It produces only offline assets — the §8.1 forbidden-runtime-output check is enforced on the CLI path — and is deterministic (same inputs → strictly byte-identical outputs; the CLI leaves the library's checksum-excluded `build_timestamp` evidence field unset).
- Wire `verify_download_bbox_matches_registry` into the direct-grid **producer preflight**: before any direct-grid production side effect (the producer performs no raw downloads — those are issued upstream by `workers/data_adapters`, a stage shared with legacy IDW basins and out of scope here), the producer resolves the registered snapshot from the verified contract identity via the registry's supersession-aware current-version query, fails closed when no snapshot resolves or when the snapshot is superseded (`superseded_at` non-NULL — the producer-preflight half of the `grid-drift-lifecycle` §"Consumers of a superseded snapshot fail closed" cross-change contract), and verifies the deployment bbox (`NHMS_DOWNLOAD_BBOX_*`, defaults in `workers/data_adapters/region.py:21-24`) matches the registered snapshot's bbox, failing closed on mismatch. Includes the longitude-convention (0..360 vs -180..180) responsibility landing that `canonical-source-grid-registry` Task 3.2 handed off.
- **Fix the dangling spec reference** in `direct-grid-binding-artifact`: repoint the station-`z` `z_policy` authority from the descoped readiness solver audit to this change's committed `z_policy` verdict evidence file (delta only; `openspec/specs/` is not edited directly — the delta applies at archive time).

## Capabilities

### New Capabilities
- `z-policy-verdict`: A narrow, three-question solver-audit verdict for the station-`z` `z_policy`, committed as an evidence file whose value (∈ `{sentinel, model_dem_at_cell_center, canonical_orography}`) is the single authority for the mapping builder's `ZPolicy` input — resolved against a pinned location + pinned SHA-256 and consumed through a deterministic, total `per_cell_z` sampling contract — with fail-closed rules when the verdict is missing, fails the pinned-checksum/value match, is out of range, or is a `sentinel` chosen without proof that `Z` is unused.
- `mapping-builder-cli`: An offline, deterministic CLI that chains the existing mapping-builder library stages (G0–G5) end to end, reads only object-store release-frozen packages (configurable root, evidence-recorded overrides), and produces only offline assets (zero production writes, §8.1 enforced on the CLI path).
- `producer-bbox-preflight`: A fail-closed producer preflight that resolves the registered grid snapshot via the registry's supersession-aware current-version query (missing or superseded snapshot → fail closed, per the `grid-drift-lifecycle` cross-change contract) and verifies the deployment download bbox matches it before any direct-grid production side effect, with an explicit longitude-convention responsibility landing.

### Modified Capabilities
- `direct-grid-binding-artifact`: Repoint the station-`z` `z_policy` authority from the descoped `cmfd-direct-grid-platform-readiness` solver audit to this change's `z_policy` verdict evidence file (single `## MODIFIED Requirements` delta; no behavior change to the tolerance/coordinate rules themselves).
